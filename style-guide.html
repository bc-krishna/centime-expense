Centime Expense

Design System & Style Guide v1.0

On-brand with Centime's teal identity. Clean, dense, task-oriented. Built for finance professionals who process hundreds of transactions.

Design Principles

Density over decoration

Tables, not cards. Compact rows, not spacious tiles. Information density is a feature.

Color = meaning

Teal for brand actions. Gold for highlights. Status colors for feedback. Nothing decorative.

Progressive disclosure

Show the summary. Let users drill in. Don't front-load every detail.

Keyboard-first

Power users live in the keyboard. Tab, Enter, shortcuts. Mouse is optional.

Do / Don't

Do
  • Use teal #008085 for primary actions only
  • Use dark teal #024743 for sidebar/nav backgrounds
  • Use gold #fdc231 sparingly for highlights and card nudges
  • Use tables for list data — compact and scannable
  • Right-align monetary amounts with tabular nums
  • Surface policy info before the user violates it
  • Use 12px border-radius on cards (Centime's style)
Don't
  • Use blue as primary — we're teal, not blue
  • Use card grids for tabular data (wastes space)
  • Add decorative gradients or illustrations
  • Mix teal and gold in the same button
  • Animate transitions beyond 150ms
  • Use color without an accompanying text label
  • Use sharp corners (4px) — Centime uses rounder shapes

1. Architecture Overview

Centime Expense follows a layered, cloud-native architecture designed for horizontal scalability, multi-tenant isolation, and white-label customization. The system is built on AWS with a React frontend, Node.js BFF, and Spring Boot microservices core.

Design Principles
Cloud-Native

AWS-native services, containerized workloads, infrastructure as code with Terraform.

API-First

OpenAPI 3.0 contracts, versioned endpoints, consistent REST + event-driven patterns.

Multi-Tenant

Shared infrastructure with logical tenant isolation via RLS, tenant-scoped caching, and per-tenant configuration.

White-Label

Fully customizable branding: logos, colors, domains, email templates, and mobile app theming per tenant.

Horizontally Scalable

Stateless services with auto-scaling, read replicas, and distributed caching for elastic throughput.

ADA Compliant

WCAG 2.1 Level AA across all interfaces, automated a11y testing in CI, screen reader optimized.

2. Frontend
Tech Stack
React 18 TypeScript 5 Vite 5 Tailwind CSS 3 Radix UI TanStack Query v5 Zustand React Hook Form Zod React Router v6 Recharts date-fns Vitest Playwright Storybook axe-core
State Management
  • Server state: TanStack Query v5 — caching, deduplication, background refetching, optimistic updates
  • Client state: Zustand — lightweight stores for UI state, tenant config, user preferences
  • URL state: React Router v6 search params for filters, pagination, and deep linking
Forms & Validation
  • React Hook Form with Zod schema validation for type-safe forms
  • Dynamic field arrays for line items, attendees, split coding
  • Server-side validation errors mapped to form fields
  • Auto-save drafts with debounced persistence
Real-Time
  • WebSocket via BFF for live notifications, approval updates, card transaction alerts
  • Automatic reconnect with exponential backoff
  • Server-sent events fallback for environments blocking WS
Mobile Strategy
  • Phase 1: Progressive Web App (PWA) — offline support, camera access, push notifications
  • Phase 2: React Native with shared business logic and native camera/GPS integration
  • Shared design tokens and component library across platforms
Accessibility
  • axe-core automated scans in CI on every PR
  • ARIA landmarks, live regions, focus management
  • Full keyboard navigation for all workflows
  • Minimum 4.5:1 contrast ratio, reduced motion support
Testing
  • Unit: Vitest + React Testing Library — component and hook tests
  • E2E: Playwright — critical path tests across browsers
  • Visual: Storybook + Chromatic — component documentation and visual regression
  • Coverage target: 80% statements, 90% critical paths
3. Backend for Frontend (BFF)
Node.js 20 LTS Fastify 4 TypeScript ECS Fargate
Responsibilities
Authentication Gateway

JWT validation via Cognito, session management, token refresh, CSRF protection.

Tenant Resolution

Extract tenant from subdomain/header, inject tenant context into all downstream calls.

API Aggregation

Compose responses from multiple microservices into frontend-optimized payloads.

Rate Limiting

Per-tenant and per-user rate limits via Redis sliding window counters.

WebSocket Hub

Manage persistent connections for real-time notifications, approval events, card alerts.

Response Shaping

Transform middleware responses, apply field-level permissions, paginate, and localize.

File Upload Proxy

Pre-signed S3 URLs for receipt uploads, virus scanning orchestration, image optimization.

Request Logging

Structured JSON logging, correlation IDs, latency metrics, audit trail propagation.

12. Phased Delivery
Phase 1 — MVP
Infrastructure
  • Single AWS region (us-east-1), Multi-AZ deployment
  • ECS Fargate for BFF + all 9 middleware services
  • RDS PostgreSQL Multi-AZ with 1 read replica
  • ElastiCache Redis cluster, S3 buckets
  • CloudFront CDN, ALB, Route 53
  • SQS queues + SNS topics for event bus
Application Scope
  • Core expense management + receipt OCR
  • Real-time card feeds (Visa/MC via Findity)
  • Policy engine, approval workflows
  • All 5 ERP integrations via Centime GL
  • Centime AP reimbursement
  • PWA mobile, white-label, ADA compliance
  • Purchase requisitions + PO creation
  • Slack/Teams notifications
Phase 2 — Fast Follow
Infrastructure Additions
  • React Native mobile app (iOS + Android) build pipeline
  • Additional Redis sharding for increased load
  • Enhanced auto-scaling policies based on Phase 1 metrics
  • SCIM provisioning endpoint (admin-service)
Application Scope
  • Virtual cards + card controls
  • GPS mileage tracking, per diem calculator
  • Delegation, custom fields, advanced white-label
  • VAT/GST tax reclaim, payroll integration
  • Slack/Teams actionable approvals
Phase 3 — Growth
Infrastructure Additions
  • Multi-region deployment (us-west-2 secondary)
  • Cross-region RDS replication
  • Global Accelerator for latency-based routing
  • ML inference endpoint (SageMaker) for anomaly detection
Application Scope
  • AI anomaly detection for expenses
  • Multi-entity consolidated reporting
  • International currency and compliance
  • Voice entry, marketplace integrations
Phase 4 — Future
Infrastructure Additions
  • Dedicated travel-service microservice
  • GDS (Global Distribution System) API integrations
  • Real-time pricing engine with caching layer
Application Scope
  • Travel booking and itinerary management
  • Pre-trip approval workflows
  • Corporate travel policy enforcement
